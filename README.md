# MovieLens-Analytics

В этом групповом проекте мы реализовали модуль для анализа набора данных MovieLens и подготовили аналитический отчёт. Проект демонстрирует навыки объектно-ориентированного программирования, работы с данными, тестирования и создания интерактивных отчётов.

## Структура проекта

*   **datasets/** — содержит исходные и обработанные файлы данных.
    *   `ml-latest-small/` — исходный датасет (файлы: `links.csv`, `movies.csv`, `ratings.csv`, `tags.csv`).
    *   `ml-latest-small-1000/` — укороченная версия датасета (1000 записей).
*   **src/** — основной модуль проекта.
    *   `movielens_analysis.py` — основной модуль с классами для работы с данными и тестами.
    *   `chop.py` — скрипт для подготовки датасета.
    *   `movielens_report.ipynb` — итоговый аналитический отчёт в формате Jupyter Notebook.

## Лицензия и использование данных

В этом проекте используется набор данных MovieLens, предоставленный GroupLens Research (Университет Миннесоты).

> Важное замечание: данный проект является учебной работой и не преследует коммерческих целей. Мы соблюдаем условия использования данных.

**Условия использования**
* Университет Миннесоты и исследователи не гарантируют корректность данных.
* Данные могут быть использованы для исследовательских целей при условии, что не будет заявлено об их одобрении со стороны Университета или GroupLens.
* Данные (включая их модификации) могут распространяться только при сохранении этих условий лицензии.
* Использование в коммерческих целях запрещено без специального разрешения.
* Программное обеспечение предоставляется «как есть» без каких-либо гарантий.

**Цитирование**
При использовании этого набора данных в публикациях рекомендуется ссылаться на следующую статью:
> *F. Maxwell Harper and Joseph A. Konstan. 2015. The MovieLens Datasets: History and Context. ACM Transactions on Interactive Intelligent Systems (TiiS) 5, 4: 19:1–19:19.*

## 1. Подготовка данных (src/chop.py)

Для ускорения анализа мы подготовили скрипт `chop.py`, который автоматически обрезает исходные файлы из директории `datasets/ml-latest-small` до первых 1000 записей. Результат сохраняется в `datasets/ml-latest-small-1000`. Все параметры обработки захардкожены в скрипте.

## 2. Разработка модуля анализа (src/movielens_analysis.py)

Мы создали модуль, реализующий работу с данными через четыре основных класса: `Ratings`, `Tags`, `Movies` и `Links`. Каждый класс инкапсулирует логику работы с соответствующим файлом данных.

**Ключевые особенности реализации**
* Чистая архитектура: отсутствие кода в глобальной области видимости (вся логика реализована через методы классов).
* Надёжность: любое необработанное исключение аннулирует работу, что гарантирует предсказуемое поведение программы.
* Тестирование: тесты на основе PyTest интегрированы непосредственно в модуль в виде отдельного класса `Tests`.

## 3. Аналитический отчёт (src/movielens_report.ipynb)

Отчёт подготовлен в Jupyter Notebook. Это позволило нам создать интерактивный документ, где можно запускать код по частям и добавлять пояснительный текст.

**Структура и логика отчёта**
* В отчёте рассказывается история о наборе данных MovieLens в формате «вопрос — ответ».
* Для анализа используются исключительно методы из модуля `src/movielens_analysis.py`.
* В каждую ячейку с кодом добавлена магическая команда `%timeit` для оценки производительности методов.
* Отчёт сделан эмоциональным и лёгким для чтения, при этом оставаясь технически точным.

## Примеры команд для терминала

### 1. Подготовка датасета
```bash
cd src
python3 chop.py
```
*(Скрипт автоматически обрабатывает данные из datasets/ml-latest-small и сохраняет результат в datasets/ml-latest-small-1000)*

### 2. Запуск тестов (с помощью pytest)
```bash
pytest src/movielens_analysis.py
```
*(Запускает тесты, определённые в модуле `movielens_analysis.py`)*

### 3. Запуск Jupyter Notebook для просмотра отчёта
```bash
cd src
jupyter notebook movielens_report.ipynb
```
*(Открывает отчёт в браузере)*
